# Verification and Delivery Gates

Apply these gates in order. Do not create a success claim by combining evidence from different layers.

## Gate 0: Inputs

Require a source/site, a keyword, four real semantic steps, and an identified official-app target. Return `need_input` if any required input is absent.

## Gate 1: Static JSON

```bash
python3 tools/scripts/check_xiangse_schema.py --strict-requestinfo /abs/source.json
python3 tools/scripts/xbs_tool.py check-editor -i /abs/source.json
```

Pass the static gate only if schema succeeds, the artifact contains exactly one alias, and editor checks contain no high-risk item. Record medium warnings and resolve them with the official-app editor-save checks at Gate 5. `--strict` intentionally fails on medium warnings, including the required new-schema wrapper, so reserve it for triage.

## Gate 2: Fixture parser unit test

```bash
python3 tools/scripts/xbs_tool.py simulate-fixture \
  -i /abs/source.json \
  --fixtures /abs/fixtures \
  --engine auto \
  --report /abs/source.fixture.simulate.json
```

Fixture mode replays a response by step. It does not fetch the URL generated by the preceding step. Therefore:

- keep the `fetch-samples` `manifest.json`; the validator binds each directory fixture to its recorded step URL and errors on a mismatch;
- treat `url_unverified` as missing URL provenance when a fixture has no manifest metadata;
- inspect each selected item;
- inspect each next request URL;
- reject concatenated, empty, placeholder, or cross-step URLs;
- do not claim WebView, navigation, cookie, challenge, or live-site success;
- never use fixture success as the final verdict.

## Gate 3: Live network chain

```bash
python3 tools/scripts/xbs_tool.py simulate-live \
  -i /abs/source.json \
  --engine auto \
  --webview-timeout 25 \
  --keyword 都市 \
  --book-index 0 \
  --chapter-index 0 \
  --report /abs/source.live.simulate.json
```

Require all of the following:

- `schema_check.status=PASS`
- no high-risk editor finding
- `simulation_verdict.status=pass`
- `overall_verdict.status=pass`
- four step statuses equal `pass`
- content is readable and exceeds the expected minimum
- every propagated URL is one valid URL for the next semantic step

`403`, `429`, challenge, login, or missing WebView capability is `blocked`, never `pass`.
The current validator executes `parserID: DOM` with `responseFormatType: html|json`. It emits structured unsupported errors for:

- `parserID: JS`;
- `responseFormatType` values other than `html` and `json`;
- pagination via `nextPageUrl` or `moreKeys.maxPage > 1`;
- `webViewSniff` and `webViewContentRules`;
- `JSParser`, `requestJavascript`, `responseJavascript`, `requestFunction`, and `responseFunction`.

Requested WebView execution that falls back from Playwright to HTTP/JSDOM is also blocked; its runtime must be reported as `webview:fallback`, never as a WebView pass. Treat all of these as an incomplete/blocked automation lane rather than proof that the official App lacks the capability. They cannot receive `pipeline run --live` pass until the simulator implements them; manual Gate 5 checks, including the pagination edge check, do not retroactively turn that automated failure into a pass.

## Gate 4: Package and roundtrip

```bash
python3 tools/scripts/pipeline_new_source.py package \
  -i /abs/source.json \
  -o /abs/source.xbs

python3 tools/scripts/xbs_tool.py roundtrip \
  -i /abs/source.json \
  -p /abs/source.verify
```

Record the XBS SHA-256. Compare the original and round-tripped JSON semantically, including types for `weight`, `enable`, `requestFilters`, and every action field.

`pipeline_new_source.py run --live` may combine Gates 1-4. Its `PIPELINE_STATUS: pass` does not satisfy Gate 5. Treat `--import-mac` only as an artifact handoff, never as proof that search, detail, catalog, content, or editor saves work in the official app.

## Gate 5: Official app runtime

Document:

- exact app version;
- official provenance or user-confirmed official installation;
- whether the binary is unmodified;
- source alias/version imported;
- stale aliases removed or disabled.
- pre-import `sourceModelList.xbs` path, backup path, and matching SHA-256.

Quit the App before touching its source list. Import and prune helpers require a new backup path and refuse to overwrite an existing backup; `IMPORT_REQUESTED` is a handoff result, not confirmation that the user accepted the App dialog.

Verify manually in that official app:

1. Import without a crash.
2. Search the agreed keyword.
3. Open the intended result and inspect detail fields.
4. Load a real catalog and open a chapter.
5. Verify normal and pagination-edge chapters.
6. Save the editor unchanged.
7. Rename once and save.
8. Change one rule field and save.

For a reversible smoke test, quit the App after verification, restore the exact backup, compare the restored SHA-256 with the pre-import hash, relaunch once to confirm persistence, then quit. Record that restoration separately from the source's runtime result.

The bundled `Tg@TrollstoreKios.app` is a modified static-analysis sample. Do not use it to satisfy this gate.

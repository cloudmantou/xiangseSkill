## 2026-03 实战补充（必须遵守）

1. **章节字段强制双斜杠写法（核心）**
   - 若 `chapterList.list` 已定位到章节节点（即使已是 `<a>`），`title/url/detailUrl` 仍优先写：
     - `title: //text()`
     - `url: //@href`
     - `detailUrl: //@href`
   - 这是香色实测兼容写法，能规避“有标题但 url 为空”的问题。
   - 不要默认写成 `@href||//a/@href[1]||@js:...` 的复杂兜底链，容易把上下文绕乱。

2. **相对链接不要过度 JS 化**
   - 多数情况下客户端会自动补 host。
   - 仅在确认客户端没有补全时，再补 `||@js` 绝对化。

3. **书源命名与交付备注分离**
   - `sourceName` 仅保留“站点名 + 版本”语义，不追加公众号后缀。
   - 公众号信息统一写在交付备注区（`delivery_notes`）：
   - `公众号:好用的软件站`

4. **列表字段优先使用 list 上下文（尤其 detailUrl）**
   - 当 `list` 已经命中单条书籍节点时，优先使用节点内字段（例如 `//h4[contains(@class,'bookname')]/a/@href`）。
   - 不要先用页面级 `canonical/meta` 覆盖 `detailUrl`，否则容易脱离当前条目上下文。
   - 若站点存在“精确命中后直接返回详情页”，再额外补一个 `url` 字段兜底（用于后续步骤取地址）。

5. **中文搜索优先 GET + encodeURIComponent**
   - 部分客户端/内核下，`POST` 中文参数可能出现编码不一致导致结果异常。
   - 搜索请求推荐：
   - `modules/article/search.php?searchkey=${encodeURIComponent(params.keyWord)}&searchtype=all`

6. **bookWorld 分类页避免 nextPageUrl 自动连翻**
   - 分类场景优先按 `pageIndex` 单页请求：`/sort/{cid}/{page}.html`。
   - 避免配 `nextPageUrl + 超大 maxPage`，否则容易连续翻页造成“加载慢/超时/加载不出来”。
   - `maxPage` 建议按站点真实总页数上限设置，不要盲目拉大。

7. **requestInfo 里 URL 清洗避免易错正则**
   - 避免写法：`replace(/\\//g,'/')`（在 JSON + JS 组合下易出现转义歧义）。
   - 推荐写法：`String(u).split('\\\\/').join('/')`。

8. **正文先判“接口二跳”再判 webView**
   - 若页面包含“token脚本 + 正文接口”（如 `chapter.js.php -> ajax2.php`），优先走接口链路，不要默认上 `webView`。
   - 只有在接口链路不可行时，才回退 `webView + webViewJs`。

9. **接口正文要处理风控请求头与混合响应**
   - 对 `ajax2` 这类接口，常需动态请求头：
   - `X-Requested-With: XMLHttpRequest`
   - `Referer: 当前章节 URL`
   - `content` 解析禁止“看到 token 就直接空返回”；要支持“脚本 + JSON 混合串”提取正文。

10. **API-first 站点优先走 JSON 接口（66shuba 实战）**
   - 若页面明显是前端渲染（`API.get('/api/...')`），不要从页面 DOM 抠详情与正文。
   - 直接以 API 为主链路：
   - 搜索：`/api/novel/search`
   - 详情：`/api/novel/detail/{bookId}`
   - 目录：`/api/novel/catalog/{bookId}`
   - 正文：`/api/novel/chapter/{bookId}/{chapterId}`（VIP 对应 `vip-chapter`）

11. **目录与正文有效性校验（66shuba 实战）**
   - 目录里常有哨兵章节（如 `C=-10000`），必须过滤非正整型章节 id。
   - `code=0` 不等于正文有效：若内容是“网络开小差了，请稍后再试”等占位文案，按上游异常处理，不算解析成功。

12. **正文 `nextPageUrl` 必须加“同章分页守卫”（sudugu 实战）**
   - 某些站点“上一章/目录/下一页(或下一章)”共用一组导航节点，不能只靠文案 `contains(text(),'下一页')` 判断。
   - 推荐先取“右侧翻页位”链接，再在 JS 里解析当前 URL 与候选 URL：
   - `/{bookId}/{chapterId}(-{page})?.html`
   - 仅当 `bookId` 相同、`chapterId` 相同、分页号递增时继续翻页；否则返回空，防止误跳下一章。

13. **分类分页 URL 形态不要猜（sudugu 实战）**
   - 同站点不同页面可能使用不同分页规则：
   - 目录页：`p-2.html#dir`
   - 分类页：`/{cat}/2.html`
   - 必须基于真实“下一页”链接或已验证模板构造，不要默认套用 `p-2` 或 `_2`。

14. **外部搜索若为加密链路，优先做可用降级（shuhaoxs 实战）**
   - 当本域搜索不可用、首页跳到外部域（如 `rrssk`）时，先判定是否存在“关键词加密 + 结果链接加密 + JS 解密跳转”。
   - 若解密依赖 `CryptoJS + UA + 动态页面变量`，不要默认当主搜索接入。
   - 优先使用“分类页遍历 + 关键词过滤”做 fallback 搜索，先保证可用，再考虑加密链路增强。

15. **目录分页接口要防重复页误翻（shuhaoxs 实战）**
   - `index.php?action=loadChapterPage` 这类接口常见：
   - 越界页重复返回末页
   - 章节较少的书在 `page>=2` 重复返回第 1 页
   - `nextPageUrl` 不能仅依赖 `list.length > 0`，要叠加 `chapterorder` 页范围校验（如每页 `1-100`、`101-200`）。

16. **17K 实战复盘（闪退 / 解密 / 分类 / 备注）**
   - 防闪退：不要把超长混淆 WAF JS 直接塞进主链路，优先 API 或可复现降级链路。
   - 解密验收：若章节响应含 `content[].encrypt=1`，必须验证正文“解密后非空且非密文”。
   - 分类完整性：`bookWorld` 与 `requestFilters` 为交付必检项，禁止遗漏分类能力。
   - 备注规范：公众号信息写交付备注（`delivery_notes`），不再写入 `sourceName`。

## 书源开发完整流程

### 第一步：网站分析与准备

1. **使用MCP Puppeteer进行网站分析**
   ```javascript
   // 导航到目标网站
   puppeteer_navigate({"url": "https://example.com/search?keyword=测试"})
   
   // 截取页面截图
   puppeteer_screenshot({"name": "analysis", "width": 1200, "height": 800})
   
   // 获取DOM结构
   puppeteer_evaluate({"script": "document.querySelector('.book-list').outerHTML"})
   ```

2. **网站基础信息收集**
   - 主域名和协议
   - 搜索接口URL模式
   - 请求方法（GET/POST）
   - 必要的请求头信息
   - 反爬虫机制（验证码、加密等）

### 第二步：搜索功能分析

1. **搜索请求分析**
   - 搜索关键词参数名
   - 分页参数（page、pageSize等）
   - 筛选参数（分类、状态、排序等）
   - 请求头要求（User-Agent、Referer等）

2. **响应格式识别**
   - JSON格式：需要设置 `responseFormatType: "json"`
   - HTML格式：使用XPath解析
   - 混合格式：可能需要JSParser处理

### 第三步：XPath规则编写

1. **基础配置要点**
   - `sourceName`: 书源名称，建议使用网站名称
   - `host`: 网站主域名，如 `https://www.shuqi.com`
   - `weight`: 权重值，影响搜索优先级，一般设置为 `9999`
   - `enable`: 启用状态，设置为 `1`
   - `sourceType`: 本仓文本书源固定为 `text`；`novel/comic` 不属于当前交付契约
   - `sourceUrl`: 书源网址
   - `lastModifyTime`: 最后修改时间戳

2. **XPath语法核心规则**
   - 双斜杠转义：`//` 需要写成 `\/\/`
   - 属性获取：`@href`、`@src`、`@title`
   - 文本获取：`text()` 或直接省略
   - 章节字段兼容写法：`title/url/detailUrl` 优先 `//text()`、`//@href`、`//@href`
   - 多选择器：使用 `||` 分隔备选方案
   - 条件筛选：`[contains(@class,'book')]`
   - 文本替换：`replace('旧文本','新文本')`

### 第四步：搜索配置实现

1. **请求信息 (requestInfo)**
   - 支持 `@js:` 前缀的JavaScript动态配置
   - 返回对象包含：`url`、`POST`、`httpParams`、`httpHeaders`
   - 支持缓存时间设置：`cacheTime`
   - 示例：
   ```javascript
   "@js:\nlet hp = {\n    'keyword': params.keyWord,\n    'page': params.pageIndex\n};\nreturn {'url': config.host + '/search', 'POST': false, 'httpParams': hp};"
   ```

2. **响应格式 (responseFormatType)**
   - `json`: JSON格式响应
   - `html`: HTML格式响应（默认）

### 第五步：列表解析配置

1. **基础字段**
   - `list`: 书籍列表容器的XPath
   - `bookName`: 书名的XPath
   - `author`: 作者的XPath
   - `cover`: 封面图片的XPath
   - `detailUrl`: 详情页链接的XPath
   - `desc`: 简介的XPath
   - `category`: 分类的XPath
   - `status`: 状态的XPath
   - `wordCount`: 字数的XPath
   - `lastChapterTitle`: 最新章节的XPath

2. **高级配置**
   - `nextPageUrl`: 下一页链接
   - `moreKeys`: 包含 `pageSize`、`maxPage`、`requestFilters` 等
   - 多重备选策略：`"bookName||title||name"`

### 第六步：章节内容解析

1. **JSParser 历史用法**
   - 仅在官方 App 已验证需要旧回调链时使用。
   - 不得仅凭修改逆向样本中的资源或符号，宣称官方规则运行时提供 AES、Base64 或浏览器全局函数。
   - 优先使用标准字段与已验证接口，避免把复杂解密脚本直接作为主链路。

2. **内容解码函数**
   ```javascript
   function functionName(config, params, result) {
       // 解密或处理逻辑
       let content = JSON.parse(result.content);
       return content.data.text;
   }
   ```

3. **加密解密处理**
   - 不假定官方 App 的规则上下文内置 `CryptoJS`、`atob` 或修改样本中的 `crypto.min.js`。
   - `LCJSTool` 符号和 `responseDecryptType=encryptType1` 只构成静态兼容线索；具体调用路径必须在官方 App 中验证。
   - 无法同时通过 live 与官方 App 明文正文验收时，返回 `blocked/fail`，不得交付密文或空正文。

### 2. 搜索配置核心规则

#### 方法1：替换规则（简单直接）
- 适用场景：网站有标准的搜索URL格式
- 配置格式：`https://网站.com/search?key=%@keyWord&p=%@pageIndex`
- 支持变量：
  - `%@keyWord` - 搜索关键词
  - `%@pageIndex` - 页码
  - `%@offset` - 偏移量
  - `%@filter` - 筛选器

#### 方法2：JS动态配置（灵活强大）
- 适用场景：复杂搜索逻辑、POST请求、特殊参数处理
- 必须以 `@js:` 开头
- 内置变量：`params`、`config`、`result`
- 返回对象支持字段：
  ```javascript
  return {
    url: "请求地址",
    POST: false,  // 是否POST请求
    httpParams: {}, // 请求参数
    httpHeaders: {}, // 请求头
    forbidCookie: false, // 禁用Cookie
    webView: false, // 使用WebView
    webViewJs: "", // WebView执行JS
    webViewJsDelay: 1, // 延迟执行
    webViewSkipUrls: [], // 跳过的URL
    sourceRegex: "" // 资源嗅探正则
  };
  ```

### 3. 列表解析配置

#### 标准字段映射
- `list` - 列表容器选择器
- `bookName` - 书名
- `author` - 作者
- `desc` - 简介
- `cat` - 类别
- `status` - 状态
- `wordCount` - 字数
- `lastChapaterTitle` - 最后章节标题
- `detailUrl` - 详情页地址
- `cover` - 封面图片地址

#### 分页控制参数
- `pageSize` - 每页数量，用于判断是否有更多页
- `maxPage` - 最大分页数
- `skipCount` - 跳过项数量
- `nextPageUrl` - 下一页地址（需配合maxPage使用）

#### 高级配置参数（moreKeys）
```json
{
  "pageSize": "16",
  "requestFilters": "分类\n全部::0\n现言::现代言情\n",
  "removeHtmlKeys": ["bookName", "author"],
  "skipCount": 6
}
```

### 4. 章节内容解析

#### JSParser高级用法
书旗小说使用了复杂的内容解码机制：
```javascript
function functionName(config, jsparams, result) {
  if(jsparams.lastResponse) {
    var data = JSON.parse(result);
    var cpContent = decodeCont(data.ChapterContent);
    return {'content':cpContent, 'removeHtmlKeys':'content'};
  }
  // 处理章节ID和URL构建
  var cid = jsparams.queryInfo.url.match(/cid=(\d+)/)[1];
  // 返回下一页URL或内容
  return {'nextPageUrl':url, 'autoRequestMore':true};
}
```

#### 内容解码函数
- 支持Base64解码
- 支持字符替换解码
- 支持UTF-8编码处理

## 常见问题与解决方案

### 1. 403 Forbidden错误
**原因：**
- 缺少User-Agent
- 缺少Referer
- 反爬虫机制

**解决方案：**
```javascript
// 完整的请求头配置
{
  "User-Agent": "Mozilla/5.0 (iPhone; CPU iPhone OS 14_0 like Mac OS X) AppleWebKit/605.1.15",
  "Referer": "网站首页",
  "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
  "Accept-Language": "zh-CN,zh;q=0.9",
  "Accept-Encoding": "gzip, deflate"
}
```

### 2. 搜索结果为空
**排查步骤：**
1. 检查搜索URL是否正确
2. 验证参数名称是否匹配
3. 确认请求方法（GET/POST）
4. 检查编码问题（URL编码）

### 3. 外部搜索服务
**处理策略：**
- 分析外部搜索网站的API
- 配置跨域请求
- 优先验证“是否有稳定明文详情链接”：
  - 若仅有 `onclick=toUrl(...)` 等前端解密跳转，默认判定高风险
  - 若依赖 `CryptoJS + UA + 动态变量` 才能解密详情地址，优先做站内 fallback 搜索
- 考虑稳定性和可用性：
  - 优先“可用且可维护”的降级方案（分类遍历 + 关键词过滤）
  - 外部加密链路作为增强项，独立实现与维护

## 实战案例深度分析

### 案例1：聚合漫画（JSON API类型）

**网站特点：**
- 使用JSON API接口
- 需要特定的请求头
- 支持分类筛选
- 图片需要添加后缀

**关键配置要点：**

#### 4. XPath解析规则详解

**XPath语法基础：** <mcreference link="https://github.com/ckstc/XsRead?utm_source=chatgpt.com" index="0">0</mcreference>
- `//` 用于定位类型（从任意位置开始搜索）
- `/` 用于表示下级关系（直接子元素）
- `()` 用于括取，可以用`(div[@id='xxx'])[3]`表示并列的id为xxx的div格式的第四个（下标从零开始）
- `div`, `img` 等对应`<div>`, `<img>`类型
- `@` 用来定位class或者id属性
- `@src` 用于获取`<div src="xxx">`里面的src属性值

**重要：香色闺阁App XPath格式要求**
- 香色闺阁App中的XPath表达式**不使用相对路径的点号（.//）**
- 必须使用**绝对路径格式（//）**
- 例如：使用`//h2/a/text()`而不是`.//h2/a/text()`

**小说书源XPath配置示例：**
```javascript
// 搜索结果列表解析（香色闺阁App格式）
"list": "//div[@class='book-list']//li",
"bookName": "//h3/a/@title",
"author": "//p[@class='author']/text()",
"desc": "//p[@class='desc']/text()",
"detailUrl": "//h3/a/@href",
"cover": "//img/@src"

// 章节列表解析（香色闺阁App格式）
"list": "//div[@class='chapter-list']//a",
"title": "//text()",
"url": "//@href",
"detailUrl": "//@href"

// 章节内容解析（香色闺阁App格式）
"content": "//div[@class='content']//p/text()"
```

**XPath调试技巧：** <mcreference link="https://github.com/UFOAlastor/xiangseguige" index="1">1</mcreference>
- 使用浏览器F12开发者工具
- 在Console中使用`$x('//xpath表达式')`测试XPath
- 确保XPath路径的唯一性和准确性
- 正则表达式后处理：XPath获取原始文本后，使用正则表达式进行精确提取
  - 例如：XPath获取"都市小说 作者：张三"，用正则`作者：(.+)`提取"张三"
  - 书名提取：用正则`^(.+?)\s+作者：`从完整文本中提取纯书名

#### 5. 技术难点与解决方案

**403 Forbidden错误：**
- 原因：缺少必要的请求头
- 解决：添加完整的User-Agent和Referer
- 预防：使用真实浏览器的请求头

**XPath解析失败：**
- 原因：XPath路径错误或网站结构变化
- 解决：使用浏览器开发者工具重新分析DOM结构
- 预防：编写更通用的XPath表达式

**搜索结果为空：**
- 原因：参数名错误或缺少隐藏参数
- 解决：仔细分析表单的所有input元素
- 预防：实际测试表单提交过程

**分页功能异常：**
- 原因：分页参数识别错误
- 解决：分析分页链接的URL模式
- 预防：测试多页搜索结果

**动态内容加载：**
- 原因：网站使用JavaScript动态加载内容
- 解决：先判断是否可走“接口二跳”（token -> ajax/json）；不可行再用 `webView`
- 示例（接口二跳）：`chapter.js.php` 提 token，再请求 `ajax2.php`
- 示例（webView回退）：`@js: return {'url':result, 'webView':true};`；仍需 live 与官方 App 同时验证，不能凭字段存在判定可用
